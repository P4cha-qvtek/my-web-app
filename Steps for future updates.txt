### When you pull from update in GitHub:

## Local:
git add .
git commit -m "Update description"
git push origin main

## EC2:
- SSH into EC2
ssh -i ~/my-web-app-key.pem ubuntu@3.214.199.76

- Navigate to project
cd /home/ubuntu/my-web-app

- Pull latest code (if using Git)
git pull origin main

- Activate venv
source venv/bin/activate or .\env\scripts\activate.bat

- Run migrations (if models changed)
cd website
python manage.py migrate

- Collect static files (if CSS/JS changed)
python manage.py collectstatic --noinput

- Restart Gunicorn
sudo systemctl restart gunicorn



-----------------
### When you update DATABASE/MEDIA:

## Local:
cd website
tar -czf db-update.tar.gz db.sqlite3 media


## WSL:
scp -i ~/my-website-server-key.pem db-update.tar.gz ubuntu@3.214.199.76:/home/ubuntu/

## EC2:
sudo systemctl stop gunicorn
cd /home/ubuntu
tar -xzf db-update.tar.gz
cp db.sqlite3 /home/ubuntu/my-web-app/website/
cp -r media/* /home/ubuntu/my-web-app/website/media/
sudo chown ubuntu:www-data /home/ubuntu/my-web-app/website/db.sqlite3
sudo chown -R ubuntu:www-data /home/ubuntu/my-web-app/website/media
sudo chmod 664 /home/ubuntu/my-web-app/website/db.sqlite3
sudo chmod -R 755 /home/ubuntu/my-web-app/website/media
sudo systemctl start gunicorn

## Backup Your Database Regularly on EC2:

cp /home/ubuntu/my-web-app/website/db.sqlite3 ~/backups/db-$(date +%Y%m%d).sqlite3
